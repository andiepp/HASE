# ADR-0006: Descriptor Resolution

## Status

Accepted

## Context

Some endpoints can provide complete descriptors directly.

Resource-constrained endpoints may not have enough memory to store or transmit
full descriptors.

HASE must support both large devices such as ESP32-based systems and very small
microcontrollers.

## Decision

HASE depends on resolved descriptors, not on how descriptors are obtained.

A descriptor may be:

- embedded in the endpoint
- referenced by descriptor id
- loaded from a repository
- generated by code
- cached locally
- provided by a simulation endpoint

The runtime only consumes resolved descriptors.

## Consequences

Advantages

- supports very small microcontrollers
- keeps the runtime independent of descriptor acquisition
- allows descriptor repositories
- supports virtual endpoints
- preserves descriptor-driven architecture

Disadvantages

- descriptor resolution requires additional services
- descriptor references must be versioned carefully
